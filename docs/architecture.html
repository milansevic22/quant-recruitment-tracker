Architecture

Firebase-First MVP

The app uses Next.js for the UI and server routes, Firebase Firestore for persistence, and Vercel for deployment.

System Overview

Layer Responsibility
Next.js App Router Dashboard page, API routes, Vercel-compatible runtime.
React components Summary cards, jobs table, companies list, status states.
Firebase client SDK Browser-safe Firestore reads for dashboard data.
Firebase Admin SDK Privileged server-side writes for seed and scan routes.
Scanner module Gentle public-page fetches, keyword matching, deduping.

Runtime Boundaries

  • The browser never receives Firebase private key material.
  • Admin routes require ADMIN_API_SECRET.
  • Dashboard reads fall back to sample data when Firebase is absent.
  • Scanner failures are saved in scan-run records for review.

Future Architecture Extension

Vercel Cron can call the scan route on a schedule once production environment variables are configured. Resend can later notify users when new jobs are added, but it is intentionally outside the core MVP loop.
