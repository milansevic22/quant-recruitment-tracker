Deployment Notes

Vercel and Firebase Setup

The app is designed to deploy on Vercel with Firebase environment variables and no separate server.

Local Setup

  1. Install dependencies with npm install or pnpm install.
  2. Copy .env.example to .env.local.
  3. Add Firebase browser and Admin SDK variables.
  4. Run npm run dev or pnpm dev and open the local app URL.

Required Environment Variables

NEXT_PUBLIC_FIREBASE_API_KEY=
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=
NEXT_PUBLIC_FIREBASE_PROJECT_ID=
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=
NEXT_PUBLIC_FIREBASE_APP_ID=

FIREBASE_PROJECT_ID=
FIREBASE_CLIENT_EMAIL=
FIREBASE_PRIVATE_KEY=
ADMIN_API_SECRET=
CRON_SECRET=
RESEND_API_KEY=
ALERT_EMAIL_TO=
ALERT_EMAIL_FROM=

Vercel Setup

  1. Create a Vercel project from the GitHub repository.
  2. Add all required environment variables in Vercel settings.
  3. Deploy with the default Next.js build settings.
  4. Call the protected seed route after the first deployment.

Protected Route Calls

Local seed:

curl -X POST http://localhost:3000/api/seed \
  -H "x-admin-secret: your-secret"

Local scan:

curl -X POST http://localhost:3000/api/scan \
  -H "x-admin-secret: your-secret"

Production calls use the same headers with the Vercel domain.

curl -X POST https://your-domain.vercel.app/api/seed \
  -H "x-admin-secret: your-secret"

Automatic Scanning

The project includes vercel.json with a daily scheduled scan at 0 13 * * *, which runs at 13:00 UTC. Vercel invokes /api/cron/scan with a GET request.

  1. Add CRON_SECRET in Vercel environment variables.
  2. Add Firebase Admin variables so the scanner can write to Firestore.
  3. Redeploy after adding vercel.json and environment variables.
  4. Open the Vercel project's Cron Jobs settings to confirm the job is active.
  5. Use Vercel function logs to review scan results and errors.

Email Alerts

Resend alerts are optional. To enable them, add RESEND_API_KEY and ALERT_EMAIL_TO in Vercel, then redeploy. ALERT_EMAIL_FROM can be set if a verified sender domain is available.

Secrets

The Firebase private key and admin secret must never be committed. Browser-safe Firebase config values use NEXT_PUBLIC_. Admin-only values do not.

Cron Status

Cron configuration is included. It becomes live after the project is redeployed on Vercel with CRON_SECRET and Firebase credentials configured.
