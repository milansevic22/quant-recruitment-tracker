Build Sequence

  1. Create HTML documentation and README.
  2. Scaffold Next.js App Router with TypeScript and Tailwind.
  3. Add Firebase client and Admin SDK setup.
  4. Define TypeScript models and seed data.
  5. Build the dashboard UI with fallback data.
  6. Add protected seed and scan API routes.
  7. Document environment variables and deployment steps.
  8. Run linting and production build checks.

Quality Checks

  • Dashboard loads without Firebase credentials.
  • Build completes without real secrets.
  • Admin routes reject missing or invalid secrets.
  • Scanner records errors without blocking the whole run.

Decision Principles

  • Use Next.js routes rather than a separate backend.
  • Keep Firebase boundaries easy to replace or test.
  • Prefer readable code over clever abstractions.
  • Document limitations directly instead of hiding them.

Known Risk Areas

Careers pages frequently use third-party systems and client-rendered listings. The MVP scanner should be treated as best-effort and paired with manual review for high-value firms.
