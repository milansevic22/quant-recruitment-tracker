Retrospective

Tradeoffs and Next Steps

A candid record of what was intentionally simple, what worked, and what should improve after the MVP.

What Worked

  • The documentation-first sequence kept product scope, architecture, and deployment expectations explicit.
  • The dashboard remains useful without Firebase credentials because it falls back to realistic sample data and supports review-mode interactions.
  • The automation center makes the monitoring loop visible to a non-technical reviewer.
  • Admin-only writes are isolated behind protected Next.js routes.
  • Scheduled scanning is wired through a secured Vercel Cron route.
  • The scanner records failures per company, which keeps partial scans reviewable.

Intentionally Simple

  • Single dashboard page rather than a multi-route admin system.
  • Cron is configured, but it only becomes live after Vercel environment variables are added.
  • Simple keyword matching instead of ranking or NLP.
  • Best-effort public-page parsing only.
  • Status changes are local in review mode until authenticated writes are added.
  • The review scan is intentionally local sample behavior until Firebase credentials are configured.

Expected Limitations

  • Some careers pages use dynamic rendering or third-party boards.
  • Locations may be incomplete when not visible in link text.
  • Manual verification remains necessary before applying.
  • Notification support is left for a later iteration.
  • Firestore rules still need to be configured for the chosen review or production access model.

Next Improvements

  • Review Vercel Cron logs after the first scheduled production scan.
  • Add Resend alerts for newly discovered jobs.
  • Add authenticated status updates for applied and ignored jobs.
  • Persist the review-scan workflow through Firestore once Firebase credentials and rules are installed.
  • Add per-company adapters for high-value job boards that expose stable APIs.
  • Add filters for role type, location, company, and status.

Final Status

The MVP builds successfully and is ready for Firebase credentials, seeding, manual scans, and Vercel deployment.
